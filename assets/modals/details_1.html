01 / PROJECT CONTEXT

多场景并行的交付危机

某大客户（工业制造领域）需在极短周期内并行构建多套智能化管理系统，业务边界横跨设备实时监控、高密文件管理、数字化质量分析、以及全天候告警中心。

传统模式痛点： 各业务项目各自为战，视觉组件碎片化，前端重复开发率居高不下，且大模型由于缺乏标准上下文，生成的代码可用度极低。
破局创新思路： 跳出"画图+手写代码"的传统怪圈。通过PM的需求分析卡口，将功能点像素级映射到一套高内聚、易注入的《中天UI规范》中，打通 Vibe Coding 自动化前端交付管线。
02 / DESIGN ARCHITECTURE

构建高内聚的组件Spec规范体系

拒绝无逻辑的散装样式。统一规范包含设计令牌（Tokens）、全场景页面布局及高复用组件库，一次定义，多项目共享。

1. 精密工程级 Design Tokens

严格推行基于 8pt Grid 间距步进系统。主色锚定中天高科技蓝（`#004EA1`），强调色使用工业安全活力橙（`#FF8D1C`），圆角规则从按钮控件（4px）到弹窗面板（8px）分级严密定义。

--primary-color: #004EA1;
--space-gap-sm: 12px;
--border-radius-card: 8px;

2. 双端兼顾的标准侧边栏布局

针对办公环境采用 200px 标准左侧三段式布局；针对工厂车间的工业触摸屏设备，规范特别开辟了 72px 窄侧边栏触控模式（仅显示大图标+短标签，防止误触）。

3. 指标卡片与 ECharts 级联

数据可视化层摒弃了随意的图表样式，统一规范了统计数字卡片矩阵（Metric-Grid）、设备率折线图、缺陷分布饼图及 CPU 实时仪表盘的配色与主题。

chart.setOption({
  colorStops: ['#004EA1', 'rgba(0,78,161,0)']
});
03 / REQUIREMENT TO DESIGN

高精准需求分析与"无缝组件映射"

向产品经理（PM）转型的关键一步：拒绝含糊不清的需求描述，建立一套从业务调研到组件映射的严密转化链条。

"每一个功能点，都在规范文件中拥有唯一的组件解。"

PM 结构化业务需求调研方法

1. 业务调研 (Pain Points) 与客户及车间一线深度沟通核心痛点（如：设备异动响应滞后、技术资料检索混乱）。
2. 场景梳理 (User Persona) 精准识别典型用户角色（如：现场运维人员、系统管理员、厂区质检员）的操作路径。
3. 功能拆解 (Priority Slicing) 按核心业务模块精细化划分功能点，严格标注优先级（P0核心 / P1次要 / P2延伸拓展）。
4. 交互定义 (Feedback Loop) 清晰、定量定义每个关键操作的触发方式、反馈延迟阈值以及全局异常兜底逻辑。

1. 结构要求：必须包含顶部面包屑搜索栏、P0级新增文件按钮、标准文件数据表格（包含文件名、类别、上传时间、状态标识、操作图标列）、以及下方的标准分页组件。

思考沉淀：一次定义，长久赋能

引入 UI规范体系 + Vibe Coding 交付链路后，设计规范成为了 AI 的"硬性逻辑铁轨"。我一个人在承担 PM 需求调研的同时，通过精密提示词在 3 天内完成了全部子系统高保真原型的快速构建与极速迭代，实现了效能的代际跨越。
